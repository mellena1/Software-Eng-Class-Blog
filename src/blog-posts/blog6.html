Week 6 - Writing Out the Stubs

by Andrew Mellen

Posted on February 21, 2019

This week we focused on getting our stubs in the backend implemented. It has been going pretty well. I started to take a backseat, as I did most of the setup for getting the stubs made. I have been reviewing every one elses' code, making sure that everything is written well, as I have the most experience with Go. This also helps us maintain code standards to make sure we all do everything similarly.

We also decided this week to change our database schema so that we could include a time slot table. This is because of the request by the client that the time slot for one session should change the rest of the session's time slots. This was not much of a change, as our backend code is modular enough to make changes very easily.

Our frontend team were able to get the main pages of phase one made, so they are now ready to start writing code to implement with our API. Our backend team needs to focus on getting everything implemented that the frontend team will need to complete their goals. This involves getting the GetAll functions done, as those are probably the most important ones to the frontend team.

All in all, I think we are on track to have a working product by next week. We aimed for completing everything by this week, but we were not able to accomplish that goal. I am confident in our ability to finish phase one by this week though.
