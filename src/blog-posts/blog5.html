Week 5 - Writing the Backend Framework

by Andrew Mellen

Posted on February 15, 2019

This week, our team met up on Monday to discuss what we had done so far and what needed to be done. It was a good time for the frontend and backend teams to have a meeting of the minds and be able to figure out how we could reach our goal of having working code within the next week or two. We decided that the frontend team should get the UI set up, and that the backend team can start implementing the API calls that the frontend team will need to call.

After our big group meeting was done, I met with the backend team so that we could make our plan for implementing the backend. I showed off what I had done for the backend framework so far, as I took the role of helping to design the code. I took some of the feedback from the group and fixed some of their concerns, and then made a pull request with my code. I think this is a big step in accomplishing our overall goal, so I am happy with that outcome.

I plan on taking on a reviewer role for the next week, so that the rest of my backend team can learn the codebase better. I will step in where necessary, but everyone seems eager to start getting the code written. I learned a lot about how to better write facades while implementing the stubs of code, so that was a positive for this week. I found it to be a really interesting design pattern in Go.

I think all in all our team is on a very good path, and that we are setting achievable goals for ourselves. I expect to have a working product within the next week or two easily.
