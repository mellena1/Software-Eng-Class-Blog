Week 4 - Organizing the Team

by Andrew Mellen

Posted on February 8, 2019

This week, our group got together to start designing our application for phase one. Our goal was to get some of the basic structure together so that we could have some direction before starting to write code. To start this process, we broke into different groups for the backend and frontend. I ended up joining the backend team, because I think my background and knowledge would be more useful to the backend team.

We decided to go with a microservice approach to make our development easier, and allow for more freedoms between the frontend and backend team. A lot of well known companies in industry are leaning towards microservices as well, so everyone on our team having this experience will be a positive.

For the backend team, we made a UML diagram to describe our basic class structure, and also an ERD to describe our database setup. For our database, we decided to go with MySQL because it is free and we have all used it before. We also decided on Go as our language of choice for the API.

Along with figuring out our class structure, I started stubbing out some of the code to get more of an idea of how our code will look. This action ended up being very productive because it exposed some of the problems with my original class diagram. I was able to switch up some of the layout to make the code cleaner and make more sense.

Overall, I think this was a productive week and sets us up to be able to meet our overall goal of having a working version of our application for phase one in the next two weeks.
